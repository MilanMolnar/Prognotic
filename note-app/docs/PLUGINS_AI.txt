[WIZARD_CODEGEN_POLICY]
These are wizard safety/product rules, not claims that the trusted CommonJS runtime cryptographically enforces them.
ALLOW=self-contained index.cjs; host API; standard JS language/Date/JSON; no dependencies
FORBID=require/import/dynamic import; fs/path/os/process/global/Buffer; child processes; network/fetch/WebSocket; eval/new Function; native modules; npm dependencies; extra files; background timers; custom UI/DOM; renderer code; cross-plugin or non-owned block access; credentials/secrets; marketplace/install/update code
REFRAME custom dashboard -> note-feed layout; chart -> stat-row + Markdown analysis; secret/API integration -> app-wide host.ai only or explain unsupported; external/cross-plugin data -> user-captured own blocks; marketplace request -> local wizard-generated folder
TRUST=do not auto-enable; user confirms plan before files; user enables afterward

[CODEGEN_TEMPLATE_MAPPING]
1 capture chosen => layout capture + command(input.text) creates owned block; optionally setPresence(false)
2 editable entries => Entry.editor command(input.blockId,input.content) validates owned id then writes
3 deletable entries => Entry.deleteCommand(input.blockId) deletes owned block
4 review workflow => capture/edit/AI setPresence(false); review action showWhen:"unvisited",tone:"review" calls acknowledgePresence
5 stats => stat-row items only from today|unvisited|total; no custom calculation UI
6 flat chronology => list; today/earlier => grouped-list groupBy:"today-recent"; calendar days => groupBy:"day"
7 AI action => permissions.ai:true + optional plugin ai.systemPrompt + action tone:"ai" + action.aiPrompt + command host.ai.complete with owned blockId and error branch
8 config choice => configSchema field + await host.getConfig() inside affected command; never capture secrets
9 top action => layout action, input {}; entry-specific action => Entry.actions, input {blockId}
10 every interactive layout declaration => same-named registered command

[INTERVIEW_DECISION_CHECKLIST]
Ask one adaptive question only when its answer changes plugin.json or index.cjs. Infer safe defaults. Target 5-8 total questions; one extra narrow follow-up only if necessary.
D1 scope_fit: can goal be expressed as owned Markdown entries in note-feed? If no, state v1 limit and offer bounded reframe.
D2 entry_model: what one capture represents; required heading/body format; never invent structured fields the host cannot render separately.
D3 capture: free-text prompt/label; whether new entries need review.
D4 list: flat vs today/recent vs day; body vs excerpt; timestamp/review badge.
D5 operations: edit,delete,entry actions,top actions; each action must map to an allowed input shape.
D6 review: what creates unvisited state and what action acknowledges it.
D7 stats: only today,total,unvisited; reframe other metrics as Markdown AI output or omit.
D8 AI: exact user benefit; only app-wide host.ai; prompt boundaries; mark permissions.ai only when used.
D9 config: only string/number/boolean/select and non-secret values that change command behavior.
D10 sidebar: label + one allowed icon.
D11 identity: concise unique id/folder slug; version fixed 1.0.0.
D12 final_plan: summarize entry model, layout, commands/input shapes, AI/config/review, id/folder, manual enable; require explicit Confirm.

[EXAMPLE_2_AI_REVIEW_INDEX_CJS]
'use strict'
const blockId = (input) => {
  const id = typeof input.blockId === 'string' ? input.blockId.trim() : ''
  if (!id) throw new Error('Choose an entry first.')
  return id
}
exports.activate = (host) => ({
  commands: {
    addEntry: async (input) => {
      const text = typeof input.text === 'string' ? input.text.trim() : ''
      if (!text) throw new Error('Describe the workout first.')
      const block = await host.blocks.createBlock(`# Workout\n\n${text}`, [host.categoryId])
      await host.blocks.setPresence(block.id, false)
      host.notify('Workout logged for review.', { tone: 'success' })
      return { blockId: block.id }
    },
    saveEntry: async (input) => {
      const id = blockId(input)
      const content = typeof input.content === 'string' ? input.content.trim() : ''
      if (!content) throw new Error('An entry cannot be empty.')
      await host.blocks.getMeta(id)
      await host.blocks.writeBlock(id, content)
      await host.blocks.setPresence(id, false)
      host.notify('Entry saved for review.', { tone: 'success' })
      return { blockId: id }
    },
    deleteEntry: async (input) => {
      await host.blocks.deleteBlock(blockId(input))
      host.notify('Entry deleted.', { tone: 'success' })
    },
    analyzeEntry: async (input) => {
      const id = blockId(input)
      const config = await host.getConfig()
      const result = await host.ai.complete({
        blockId: id,
        prompt: `Analyze this workout. Use a ${config.analysisStyle === 'detailed' ? 'detailed' : 'brief'} response.`,
        maxTokens: 700
      })
      if (result.error) throw new Error(result.error)
      await host.blocks.appendToBlock(id, `## Analysis\n\n${result.text}`)
      await host.blocks.setPresence(id, false)
      host.notify('Analysis added for review.', { tone: 'info' })
      return { blockId: id }
    },
    markReviewed: async (input) => {
      const id = blockId(input)
      await host.blocks.acknowledgePresence(id)
      host.notify('Entry reviewed.', { tone: 'success' })
      return { blockId: id }
    }
  }
})
